2. readme.xml文件解释

xxxxxxxxxx
<detail>
    <name>JsExtension</name>
    <pattern>http://.*</pattern>
</detail>

解释：

  • name是这个扩展的名字，注意命名要唯一，也是就是尽量不要和其他的Extension命名重复。否则可能会产生不可预期的错误。
  • pattern是这个扩展所使用的范围（作用域）。比如我希望这个js扩展在baidu上面使用，就可以写成http.*baidu.com.*。这个使用的就是java中的正则表达式进行匹配的。详情可以看Java 正则表达式 | 菜鸟教程 (runoob.com)

5.App提供的js接口（JsInterface）

因为这个app如果只是一个框架，而js扩展有些权限也不够，因此app为扩展提供了一个一些可以调用的接口，切勿滥用！！！

每个js接口在app中对映射成了一个对象，这个对象的名字是唯一的。调用的方式：id.function()

接口分类：

  • SET

    设置使用一个个的键值对来存放和使用的。这些设置是直接影响app的设置的，并且会保存在app中，下次启动依然有效。

  • DATA

  • FILE
